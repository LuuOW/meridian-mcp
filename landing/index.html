LLM proposes, orbital judges

Llama-3.3-70B authors fresh candidates for your task — opinionated candidate document bodies with use-it-for, workflow, heuristics, anti-patterns. The orbital engine then classifies each one deterministically.

▲

Celestial classification

Each candidate gets a class — planet, moon, trojan, asteroid, comet, or irregular — based on physics features derived from its content (mass, scope, cross-domain affinity).

⌬

Open source core

The MCP client is a 5 KB stdio shim under MIT. Same backend powers the browser miniapp and the npm package. One source of truth.

How it works

1

LLM authoring

Llama-3.3-70B (GitHub Models) writes 5 candidates for your task — full markdown bodies, ~600 chars each, with named tools and decision rules.

2

Physics signature

For each candidate we derive mass, scope, independence, cross_domain, fragmentation, drag, and dep_ratio — from content alone.

3

Class + score

Argmax over six per-class scores assigns the celestial class. Final ranking = lexical relevance × class boost × Lagrange versatility.
